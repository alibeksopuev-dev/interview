# Разбор алгоритма `deepClone` (Глубокое копирование)
> Как правильно сделать полную независимую копию объекта или массива без сохранения ссылок на оригинальные данные.

---

## 📌 Введение: что такое Deep Clone?

Термин "глубокое копирование" (deep clone) формально не определён в спецификации JavaScript, но прекрасно известен всем разработчикам. Глубокое копирование создаёт **полностью независимую копию** значения: получается новый объект в памяти, у которого **нет никаких ссылок**, указывающих обратно на оригинальные вложенные массивы или объекты. Любые изменения в глубокой копии никак не повлияют на оригинал.

**Задача:**
Написать функцию `deepClone(value)`, которая возвращает глубокую копию JSON-совместимого значения. На вход могут подаваться `null`, логические значения, числа, строки, массивы или обычные объекты (plain objects).
Алгоритм не должен обрабатывать циклические ссылки (circular references) или специальные объекты вроде `Date`, `RegExp`, `Map` или `Set`. Примитивные значения можно возвращать как есть.

### Примеры:
```javascript
const obj1 = { user: { role: 'admin' } };
const clonedObj1 = deepClone(obj1);

clonedObj1.user.role = 'guest'; // Изменяем роль в клоне
console.log(clonedObj1.user.role); // 'guest'
console.log(obj1.user.role); // Должно остаться 'admin'

const obj2 = { foo: [{ bar: 'baz' }] };
const clonedObj2 = deepClone(obj2);

obj2.foo[0].bar = 'bax'; // Изменяем оригинал
console.log(obj2.foo[0].bar); // 'bax'
console.log(clonedObj2.foo[0].bar); // Должно остаться 'baz'
```

Написать идеальное решение для `deepClone` с нуля (с поддержкой всех спец-объектов) в рамках обычного интервью почти невозможно. Обычно рамки сужаются, и интервьюеров больше интересует, как вы умеете определять типы данных и использовать встроенные методы объекта для рекурсивного обхода.

---

## ⏱️ 1. Временная сложность: `O(N)`

Где `N` — суммарное количество всех свойств, под-свойств и элементов массива во всей структуре.
Мы обязаны посетить каждый узел дерева ровно один раз, чтобы скопировать его. Быстрее `O(N)` эту задачу решить математически невозможно, так как нам нужно прочитать каждый элемент.

---

## 💾 2. Пространственная сложность: `O(N)`

В отличие от `deepEqual`, где память тратилась только на стек вызовов `O(D)`, в `deepClone` пространственная сложность строго **`O(N)`**.
Причина: мы создаём **абсолютно новые копии** всех данных в памяти. Память, требуемая для хранения нового объекта, прямо пропорциональна размеру оригинального объекта `O(N)`. (Стек вызовов тоже берёт `O(D)`, но `O(N)` физических данных всегда больше, поэтому доминирует именно `O(N)`).

---

## 🔬 3. Пошаговый разбор (Трассировка стека)

### Входные данные:
```typescript
value = { a: [1] }
```

1. **`deepClone({ a: [1] })`** (Уровень 0 - Объект)
   - `typeof value === 'object'` и `value !== null`. Базовый случай не сработал.
   - Это массив? Нет.
   - Создаём пустой объект `clone = {}`.
   - Получаем ключи через `Object.keys()`: `['a']`.
   - Цикл. Ключ `'a'`.
   - Вызываем `deepClone(value['a'])`, то есть `deepClone([1])`.

2. **`deepClone([1])`** (Уровень 1 - Массив)
   - Базовый случай не сработал.
   - Это массив? Да (`Array.isArray(value)`).
   - Вызываем `.map()` для каждого элемента. Элемент `1`.
   - Вызываем `deepClone(1)`.

3. **`deepClone(1)`** (Уровень 2 - Примитив)
   - `typeof value === 'number'`. Срабатывает базовый случай!
   - Возвращаем примитив `1` как есть (он передаётся по значению, а не по ссылке).

4. **Возврат на Уровень 1:**
   - `.map()` собирает ответы в новый массив `[1]`.
   - Массив пройден. Возвращаем новый `[1]`.

5. **Возврат на Уровень 0:**
   - В свойство `clone['a']` записывается этот новый массив `[1]`.
   - Возвращаем готовый скопированный объект `{ a: [1] }`.

**Итог:** У нас на руках совершенно новый объект с новым массивом внутри, никак не связанный с оригиналом в оперативной памяти.

## 💡 4. FAQ и разбор подходов

### Approach 1: `JSON.parse(JSON.stringify(obj))` (Худший вариант)
Самый ленивый способ, который часто предлагают новички:
```javascript
export default function deepClone(value) {
  return JSON.parse(JSON.stringify(value));
}
```
**Почему интервьюер забракует этот код?**
1. Игнорирует ключи типа `Symbol`.
2. Ключи со значением `undefined` или функциями просто **бесследно удаляются** при стрингификации.
3. Опасные мутации: объекты `Date` превратятся в строки (ISO), а `NaN` и `Infinity` превратятся в `null`.
4. Слишком ресурсоёмко: превращать гигантский объект в текст, а потом обратно парсить текст — это безумно дорого для процессора.

### Approach 2: Рекурсия (Рекомендованный)
Обходим структуру рекурсивно. Примитивы возвращаем сразу (базовый случай), массивы копируем, объекты тоже.

**Как правильно перебирать ключи объекта?**
В JavaScript есть два пути:
1. Цикл `for...in`.
2. `Object.keys()` или `Object.entries()`.

**В чём отличие?**
Цикл `for...in` перебирает не только свойства самого объекта, но и **унаследованные свойства** из его прототипа. А вот `Object.keys()` и `Object.entries()` берут **только собственные (direct) свойства** объекта. 
При клонировании обычно хотят скопировать только собственные свойства, поэтому использование `Object.keys()` — это правильный (и безопасный) путь.

В нашем файле `.ts` мы используем оптимальный вариант с `Object.keys()` и обычным циклом `for`, чтобы не создавать мусорные массивы `[[k, v]]`, как это делает красивый, но медленный `Object.entries()`.

## 🚀 6. Deep Clone II (Продвинутый уровень)

Базовое решение отлично подходит для 40-минутного интервью. Но что если нас попросят сделать "идеальный" клон, который учитывает спец-объекты, сохраняет прототипы и не ломается на циклических ссылках?

Вот какие механизмы нам нужно задействовать:

1. **Точное определение типа:** Обычный `typeof` скажет, что `Array`, `Date`, `Map` и `Set` — это просто `'object'`. Мы используем `Object.prototype.toString.call(value)`, чтобы получать точные теги вроде `[object Date]`.
2. **Кэш (Map) против циклических ссылок:** Перед тем как нырнуть в рекурсию для объекта, мы сохраняем его в `Map` (ключ = оригинал, значение = его свежий клон). Если мы снова встретим этот же объект внутри него самого, мы просто вернём готовый клон из `Map`, прервав бесконечный цикл (Stack Overflow).
3. **Сохранение прототипов:** Вместо создания пустого объекта `{}` мы делаем `Object.create(Object.getPrototypeOf(value))`. Это гарантирует, что копия унаследует те же методы класса, что и оригинал.
4. **Скрытые и Symbol-ключи:** Вместо `Object.keys()` мы используем `Reflect.ownKeys()`. Этот метод видит абсолютно всё: и обычные строки, и ключи-символы (`Symbol`), и даже свойства с флагом `enumerable: false`.
5. **Индивидуальный подход к спец-объектам:** Для `Date` и `RegExp` мы вызываем их конструкторы `new Date(value)`. Для `Map` и `Set` мы создаём новые пустые коллекции и рекурсивно копируем каждый их элемент.

*(Полный код этого решения добавлен в файл `deepClone.ts` под названием `deepCloneAdvanced`)*.

### Нативное решение (One-liner `structuredClone`)

На сегодняшний день (во всех современных браузерах и Node.js 17+) писать собственную реализацию `deepClone` для рабочих проектов больше не нужно! В JavaScript наконец-то добавили встроенный мощный API:

```javascript
const clonedObj = structuredClone(obj);
```
`structuredClone` написан на плюсах внутри движка и умеет делать глубокое копирование почти всего: циклических ссылок, `Map`, `Set`, `Date`, `RegExp` и даже бинарных данных вроде `ArrayBuffer`.

**Единственный минус `structuredClone`:**
Он всё ещё **не умеет** копировать функции, методы классов и Property descriptors (дескрипторы свойств). Если передать объект с функцией, он выбросит ошибку `DataCloneError`. В таких исключительных случаях продвинутая ручная рекурсия — это единственный выход.

## ⚠️ 7. Edge Cases (Граничные случаи базового решения)

Текущее решение создано специально для рамок 40-минутного интервью. Оно намеренно **не обрабатывает**:
1. **Неперечисляемые и Symbol-свойства:** `Object.keys` их просто не увидит.
2. **Дескрипторы свойств:** Флаги вроде `writable: false` или `get/set` геттеры не скопируются.
3. **Циклические ссылки (Circular References):** Если объект ссылается сам на себя (`obj.self = obj`), алгоритм упадёт с ошибкой `Maximum call stack size exceeded` (бесконечный цикл).
4. **Копирование прототипов:** Связь с оригинальным `__proto__` будет потеряна.
5. **Спец-объекты:** `Date`, `RegExp`, `Map`, `Set` не будут скопированы правильно — они превратятся в пустые `{}`, так как их внутреннее состояние недоступно через обычные строковые ключи.

*(Эти граничные случаи обычно решаются в задаче продвинутого уровня: Deep Clone II).*
