# TypeScript: Базовые Utility Types (Pick, Omit, Partial, Required)

**Utility Types** (Утилитарные типы) — это встроенные инструменты TypeScript, которые позволяют быстро создавать новые типы на основе уже существующих. Это как фильтры или модификаторы для ваших интерфейсов.

Чтобы понять, как они работают, давайте возьмем один базовый интерфейс «Пользователя», который мы будем изменять:

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  avatarUrl?: string; // Необязательное поле (может не быть)
}
```

---

## 1. `Pick<Type, Keys>` (Выбрать)

**Что делает:** Берет существующий тип и "выдергивает" из него только те поля, которые вы укажете.
**Аналогия:** Вы смотрите в большое меню ресторана, но выбираете только два блюда.

**Когда использовать:** Когда компоненту или функции нужна только часть данных (например, превью карточки пользователя).

**Пример:**
```typescript
// Мы хотим создать тип только для короткой карточки (имя и аватар)
type UserPreview = Pick<User, 'name' | 'avatarUrl'>;

// Как TypeScript видит это под капотом:
// type UserPreview = {
//   name: string;
//   avatarUrl?: string;
// }

const preview: UserPreview = {
  name: 'Alibek',
  // email здесь написать не получится — TypeScript будет ругаться
};
```

---

## 2. `Omit<Type, Keys>` (Исключить)

**Что делает:** Работает как анти-Pick. Берет существующий тип и удаляет из него указанные поля. Все остальное остается.
**Аналогия:** Вы говорите официанту: "Принесите мне бургер, но *исключите* лук и помидоры".

**Когда использовать:** Когда вам нужно скопировать тип почти целиком, но убрать парочку системных или секретных полей (например, при создании сущности).

**Пример:**
```typescript
// При регистрации пользователя (создании) у него еще нет ID
type UserCreateDTO = Omit<User, 'id'>;

// Как TypeScript видит это под капотом:
// type UserCreateDTO = {
//   name: string;
//   email: string;
//   avatarUrl?: string;
// }

const newUser: UserCreateDTO = {
  name: 'Ivan',
  email: 'ivan@example.com'
  // id здесь указывать не нужно
};
```

---

## 3. `Partial<Type>` (Частичный / Необязательный)

**Что делает:** Берет тип и делает **абсолютно все** его поля необязательными (добавляет знак `?` к каждому полю).
**Аналогия:** Анкета, в которой можно заполнить любые поля по желанию, а можно и вообще ничего не заполнять.

**Когда использовать:** Идеально подходит для обновления данных (когда мы отправляем на бэкенд только то, что изменилось), или для настроек по умолчанию.

**Пример:**
```typescript
// Мы хотим обновить профиль пользователя. Мы можем поменять только имя, только email, или ничего.
type UserUpdateDTO = Partial<User>;

// Как TypeScript видит это под капотом:
// type UserUpdateDTO = {
//   id?: number;
//   name?: string;
//   email?: string;
//   avatarUrl?: string;
// }

// Можно передать только одно поле
const updateData: UserUpdateDTO = {
  name: 'Alibek (Updated)'
};
```

---

## 4. `Required<Type>` (Обязательный)

**Что делает:** Работает как анти-Partial. Берет тип и делает **абсолютно все** его поля обязательными (убирает знаки `?`, если они были).
**Аналогия:** Строгий таможенник — пока не покажешь все документы до единого, не пропустит.

**Когда использовать:** Когда вы сливаете дефолтные настройки с пользовательскими, и на выходе хотите получить объект, в котором точно есть все поля.

**Пример:**
```typescript
// В базовом User поле `avatarUrl` было необязательным (с `?`).
// Теперь мы требуем, чтобы оно точно было!
type FullyLoadedUser = Required<User>;

// Как TypeScript видит это под капотом:
// type FullyLoadedUser = {
//   id: number;
//   name: string;
//   email: string;
//   avatarUrl: string; // <-- Знак вопроса исчез!
// }

const userReady: FullyLoadedUser = {
  id: 1,
  name: 'Alibek',
  email: 'test@test.com',
  avatarUrl: 'https://img.com/1.png' // Теперь это поле обязательно к заполнению
};
```

---

## Комбинация утилит (Комбо)

В реальных проектах эти утилиты часто комбинируют, чтобы создавать точные типы.

Например, форма редактирования пользователя:
*   ID трогать нельзя.
*   Остальные поля можно менять (они необязательные).

```typescript
// 1. Убираем id (Omit)
// 2. Делаем оставшееся необязательным (Partial)
type UpdateUserForm = Partial<Omit<User, 'id'>>;

// Результат:
// {
//   name?: string;
//   email?: string;
//   avatarUrl?: string;
// }
```
