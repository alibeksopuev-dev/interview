# `this` в обычных и стрелочных функциях

Разница в поведении контекста `this` — один из самых часто задаваемых вопросов на собеседованиях. Это знание критически важно для понимания того, как работают замыкания и функции высшего порядка (например, каррирование).

---

## 1. Обычные функции (`function`)

В обычной функции значение `this` является **динамическим**. Оно определяется в момент **вызова** функции, а не в момент её создания.

*   Если вызвать функцию как метод объекта: `obj.method()`, то `this === obj`.
*   Если вызвать функцию просто так: `func()`, то `this === undefined` (в строгом режиме) или `window`.
*   Если использовать `call`, `apply` или `bind`, то `this` будет равен первому аргументу этих методов.

**Проблема:** Обычная функция "забывает" контекст родителя, если её вызвать отдельно.

```javascript
const person = {
  name: 'Alibek',
  sayHi: function() {
    setTimeout(function() {
      console.log(this.name); // Ошибка! this будет равен window или undefined
    }, 100);
  }
};
```

---

## 2. Стрелочные функции (`=>`)

У стрелочных функций **нет своего `this`**. Они используют **лексический контекст** — это значит, что они "берут" `this` из того места, где были объявлены.

*   Они не создают свой контекст.
*   Они не могут быть функциями-конструкторами (нельзя вызвать через `new`).
*   Методы `call`, `apply` и `bind` не могут изменить их `this` (они просто игнорируются для контекста).

```javascript
const person = {
  name: 'Alibek',
  sayHi: function() {
    setTimeout(() => {
      console.log(this.name); // Работает! Берет this из sayHi, который равен person
    }, 100);
  }
};
```

---

## 3. Почему это важно для каррирования?

В реализации каррирования мы часто возвращаем одну функцию из другой.

```typescript
function curry(func) {
  return function curried(this: any, ...args) {
    if (args.length >= func.length) {
      return func.apply(this, args);
    }

    // Возвращаем СТРЕЛОЧНУЮ функцию
    return (nextArg) => {
      // Благодаря стрелке, этот "this" в точности равен "this" из функции curried.
      // Если бы здесь была обычная функция, контекст бы потерялся при следующем вызове.
      return curried.apply(this, [...args, nextArg]);
    };
  };
}
```

### Главное отличие (Аналогия):

*   **Обычная функция** — это как наемный рабочий. Ему всё равно, кто его нанял раньше, он подчиняется тому, кто отдает команды **сейчас** (в момент вызова).
*   **Стрелочная функция** — это как ребенок. Он всегда помнит свою "семью" (контекст, где он родился) и носит её фамилию, куда бы его ни позвали.

---

## 📐 Шпаргалка

| Особенность | Обычная функция (`function`) | Стрелочная функция (`=>`) |
| :--- | :--- | :--- |
| **Собственный `this`** | Да, создается при вызове | Нет, берется из окружения |
| **Как определяется `this`** | Зависит от того, **как** вызвали | Зависит от того, **где** создали |
| **Использование `new`** | Можно (как конструктор) | Нельзя (выдаст ошибку) |
| **Метод `bind/call`** | Работает полностью | Меняет аргументы, но не `this` |
