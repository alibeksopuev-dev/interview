# Циклические ссылки (Circular References)
> Понятие, опасности и способы решения в JavaScript.

---

## 📌 Что это такое?

**Циклическая ссылка** возникает, когда объект ссылается сам на себя либо напрямую, либо через цепочку других объектов. Это превращает структуру данных из "дерева" в "граф с циклами".

### Примеры:

**1. Прямая ссылка:**
```javascript
const obj = {};
obj.self = obj; 
```

**2. Косвенная ссылка (через посредника):**
```javascript
const a = {};
const b = {};

a.next = b;
b.prev = a; // Образовался цикл: a -> b -> a
```

---

## ☢️ Чем они опасны?

### 1. Бесконечная рекурсия (Stack Overflow)
Большинство алгоритмов обхода объектов (например, глубокое копирование или сравнение) используют рекурсию. Если алгоритм не умеет отслеживать уже посещенные объекты, он будет ходить по кругу бесконечно, пока не переполнится стек вызовов.

```javascript
function walk(obj) {
  console.log(obj);
  Object.values(obj).forEach(walk); // Упадёт на цикле
}
```

### 2. Ошибки сериализации
Стандартный метод `JSON.stringify()` не умеет работать с циклами и сразу выбрасывает ошибку:
`TypeError: Converting circular structure to JSON`

### 3. Утечки памяти (Memory Leaks)
В старых браузерах (например, IE6) сборщик мусора, работающий на "подсчете ссылок" (reference counting), не мог очистить память от объектов, которые ссылаются друг на друга, даже если на них больше никто не ссылается извне.
*Современные сборщики мусора используют алгоритм **Mark-and-Sweep**, который успешно справляется с циклами.*

---

## 🛠️ Как бороться? (Механизм Cache/Visited)

Для безопасной работы с такими структурами используется паттерн **"Список посещенных узлов"**. В JavaScript для этого идеально подходит `Map` или `WeakMap`.

### Алгоритм:
1. Перед обработкой объекта проверяем: "Есть ли он в нашем списке `Visited`?".
2. Если **ДА** — возвращаем готовую ссылку/результат из списка (прерываем рекурсию).
3. Если **НЕТ** — добавляем объект в список и продолжаем обработку.

### Почему `WeakMap` лучше?
Если вы пишете долгоживущий кэш, лучше использовать `WeakMap`. Он хранит "слабые" ссылки на объекты. Это значит, что если объект больше нигде не используется, кроме этого кэша, сборщик мусора сможет его удалить, и кэш не станет причиной утечки памяти.

## 🚀 Примеры в реальном мире

1. **DOM-дерево:** У каждого элемента есть ссылка `parentNode`, а у родителя есть `childNodes`. Это классический пример циклической связи в браузере.
2. **structuredClone:** Нативный метод JS для глубокого копирования, который "под капотом" использует кэш и умеет копировать циклы.
3. **Redux / State Management:** Иногда в сложных состояниях сущности ссылаются друг на друга (например, `Post -> Author` и `Author -> Posts`).
