# `for...in` vs `for...of` в JavaScript / TypeScript

Если говорить совсем просто, то главное отличие в том, **что именно** они перебирают:

- **`for...in`** перебирает **ключи** (имена свойств или индексы).
- **`for...of`** перебирает **значения** (сами элементы).

Давай разберем на наглядных примерах.

---

### 1. Как они работают с массивами (Arrays)

Представь, что массив — это многоквартирный дом. Индексы — это номера квартир (0, 1, 2...), а значения — это жильцы в
этих квартирах.

```javascript
const fruits = ['🍎 Яблоко', '🍌 Банан', '🍒 Вишня']

// ❌ for...in выведет "номера квартир" (индексы)
for (const key in fruits) {
  console.log(key)
}
// Результат:
// "0"
// "1"
// "2"

// ✅ for...of выведет самих "жильцов" (значения)
for (const value of fruits) {
  console.log(value)
}
// Результат:
// "🍎 Яблоко"
// "🍌 Банан"
// "🍒 Вишня"
```

> **Правило:** Для массивов почти всегда нужно использовать **`for...of`**, так как нам обычно нужны сами элементы, а не
> их индексы. Кроме того, `for...in` может работать медленнее и иногда выдавать неожиданные результаты с массивами.

---

### 2. Как они работают с объектами (Objects)

Объекты — это наборы пар "ключ: значение".

```javascript
const user = {
  name: 'Алибек',
  age: 25,
  role: 'Developer',
}

// ✅ for...in отлично подходит для объектов. Он перебирает ключи.
for (const key in user) {
  console.log(`${key}: ${user[key]}`)
}
// Результат:
// "name: Алибек"
// "age: 25"
// "role: Developer"

// ❌ for...of НЕ работает с обычными объектами.
for (const value of user) {
  console.log(value)
}
// Результат: ОШИБКА! TypeError: user is not iterable
```

> **Правило:** Для обычных объектов используй **`for...in`** (или современные методы `Object.keys()`, `Object.values()`,
> `Object.entries()`).

---

### 🔥 Продвинутая работа с объектами (на примерах из алгоритмики / LeetCode)

Часто в алгоритмических задачах мы используем объект как **Hash Map (Частотный словарь)** для подсчета элементов. Давай
посмотрим, как перебирать такие объекты.

#### Пример 1: Использование `for...in` (Классический способ)

**Задача:** Найти символ, который встречается в строке только один раз. (По мотивам LeetCode 387: First Unique
Character)

```typescript
const str = 'aabbcdd'
const freq: Record<string, number> = {}

// Шаг 1: Заполняем частотность букв
for (const char of str) {
  freq[char] = (freq[char] || 0) + 1
}
// Сейчас freq выглядит так: { a: 2, b: 2, c: 1, d: 2 }

// Шаг 2: Перебираем объект и ищем уникальный символ
for (const key in freq) {
  if (freq[key] === 1) {
    // Обращаемся к значению по ключу
    console.log(`Уникальный символ: ${key}`) // Выведет "c"
    break
  }
}
```

#### Пример 2: Использование `Object.keys()`, `Object.values()`, `Object.entries()`

Современный JS предлагает методы, которые превращают объект в **массив**, и по ним уже можно удобно бежать с помощью
`for...of`.

- **`Object.keys(obj)`** — возвращает массив **ключей**: `['a', 'b', 'c', 'd']`
- **`Object.values(obj)`** — возвращает массив **значений**: `[2, 2, 1, 2]`
- **`Object.entries(obj)`** — возвращает массив **пар [ключ, значение]**: `[['a', 2], ['b', 2], ['c', 1], ['d', 2]]`

**Задача:** Найти самый частый элемент в массиве (По мотивам LeetCode 169: Majority Element).

```typescript
const nums = [3, 2, 3]
const countMap: Record<number, number> = {}

// Шаг 1: Считаем частоту каждого числа
for (const num of nums) {
  countMap[num] = (countMap[num] || 0) + 1
}
// countMap: { '2': 1, '3': 2 }

// Шаг 2: Нам нужно найти число (ключ) с максимальной частотой (значением).
// Воспользуемся Object.entries(), так как нам нужны и ключи, и значения сразу:
let maxFreq = 0
let majorityElement: number | null = null

// [num, freq] - это деструктуризация пары из Object.entries()
for (const [num, freq] of Object.entries(countMap)) {
  if (freq > maxFreq) {
    maxFreq = freq
    majorityElement = num
  }
}

console.log(`Самый частый элемент: ${majorityElement}`) // Выведет "3"
```

> 💡 **Совет:** Комбинация `Object.entries()` + `for...of` часто считается более элегантной и читаемой, чем `for...in`,
> потому что не нужно каждый раз писать `countMap[key]`, чтобы достать значение. Мы сразу получаем готовую пару
> `[ключ, значение]`.

---

### 3. Строки (Strings)

Строки тоже можно перебирать.

```javascript
const word = 'Hi'

// for...in переберет индексы букв
for (const index in word) {
  console.log(index) // Выведет: "0", "1"
}

// for...of переберет сами буквы (это то, что обычно нужно)
for (const letter of word) {
  console.log(letter) // Выведет: "H", "i"
}
```

### Краткая шпаргалка

| Цикл           | Что перебирает?                    | Для чего лучше использовать?                | С чем НЕ работает?        |
| :------------- | :--------------------------------- | :------------------------------------------ | :------------------------ |
| **`for...in`** | **Ключи** (имена свойств, индексы) | Для **объектов** `{}`                       | —                         |
| **`for...of`** | **Значения** (сами данные)         | Для **массивов** `[]`, строк `""`, Set, Map | С обычными объектами `{}` |

**Запоминалка:**

- **IN** — это **ИН**дексы (и ключи).
- **OF** — это значения (элементы **OF** массива).
