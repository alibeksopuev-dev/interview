# Rest vs Spread Operators и методы контекста (call, apply, bind)

В JavaScript и TypeScript символ троеточия `...` выполняет две взаимоисключающие роли в зависимости от того, где он находится. Понимание этой разницы критически важно для работы с функциями высшего порядка, такими как каррирование.

---

## 📥 1. Rest-оператор (Сборщик)

**Где используется:** Только при объявлении функции в списке параметров или при деструктуризации.
**Что делает:** Собирает оставшиеся "разрозненные" аргументы, переданные через запятую, в один полноценный массив.

```typescript
function collect(...args: any[]) {
  // Все, что передали через запятую, теперь лежит в массиве args
  console.log(args);
}

collect(1, 2, 3); // Выведет: [1, 2, 3]
```

> [!TIP]
> Rest-параметр всегда должен быть **последним** в списке аргументов.

---

## 📤 2. Spread-оператор (Распылитель / Распаковщик)

**Где используется:** При вызове функции или внутри литералов массива/объекта.
**Что делает:** "Вскрывает" массив и рассыпает его содержимое на отдельные элементы (аргументы).

```typescript
const numbers = [1, 2, 3];

// Распаковываем массив в аргументы функции
console.log(...numbers); 
// Эквивалентно: console.log(1, 2, 3)
```

---

## 🔄 3. Использование с call, apply и bind

Разница между этими методами часто завязана на том, как они принимают аргументы.

### `apply(this, [args])`
Метод `apply` ожидает массив аргументов вторым параметром. Ему **не нужен** оператор Spread, так как он сам распаковывает массив внутри себя.
```typescript
func.apply(this, [1, 2, 3]);
```

### `call(this, arg1, arg2, ...)`
Метод `call` ожидает аргументы через запятую. Если у вас есть массив, вы **обязаны** использовать Spread, чтобы передать его элементы по отдельности.
```typescript
const args = [1, 2, 3];
func.call(this, ...args); // Распаковка: превращает массив в список через запятую
```

### `bind(this, arg1, arg2, ...)`
Метод `bind` работает аналогично `call`. Он принимает аргументы через запятую для частичного применения (каррирования).
```typescript
const partialFunc = func.bind(this, ...args);
```

---

## 🛡️ 4. TypeScript: "Поддельный" параметр `this`

В TypeScript можно явно указать тип контекста `this` внутри функции, передав его первым аргументом.

```typescript
function curried(this: any, ...args: any[]) { ... }
```

**Ключевые факты:**
1. **Синтаксический сахар:** Это существует только в TypeScript для проверки типов.
2. **Удаление:** При компиляции в JavaScript этот параметр полностью исчезает.
3. **Позиция:** `this` обязан быть самым первым параметром, иначе TypeScript выдаст ошибку.
4. **Зачем это нужно:** Чтобы внутри функции вы могли безопасно обращаться к `this.prop`, и TypeScript знал, что этот проп существует.

## 📐 Сводная таблица

| Оператор | Контекст | Действие | Результат |
| :--- | :--- | :--- | :--- |
| **Rest** (`...args`) | Параметры функции | Собирает запятые | Массив |
| **Spread** (`...args`) | Вызов функции | Распаковывает массив | Список через запятую |
| **apply** | Вызов | Принимает массив напрямую | - |
| **call / bind** | Вызов | Принимают список через запятую | - |
