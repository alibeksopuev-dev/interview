# Рекурсия (Recursion) простыми словами

**Рекурсия** — это когда функция вызывает саму себя. Чтобы не уйти в бесконечный цикл и не "сломать" компьютер, у любой нормальной рекурсии должны быть две части:

1.  **Базовый случай (Условие выхода):** Точка, в которой функция говорит: "Всё, стоп, я всё сделала, дальше вызывать себя не буду".
2.  **Рекурсивный шаг:** Момент, когда функция вызывает себя снова, но с чуть-чуть другими (обычно более простыми) данными.

---

## 🪆 Лучшая аналогия: Матрёшка

Представь, что тебе нужно найти маленькую конфету, спрятанную в самой-самой маленькой матрёшке.

1.  Ты открываешь матрёшку (вызываешь функцию).
2.  **Проверка:** Есть ли внутри конфета? (Базовый случай).
3.  Если конфеты нет, но есть матрёшка поменьше — ты открываешь её (рекурсивный шаг).
4.  Ты повторяешь это, пока не дойдешь до конфеты.

---

## 🏗️ Пример на коде: Факториал

Факториал числа `n` (пишется `n!`) — это произведение всех чисел от 1 до `n`.
Например: `5! = 5 * 4 * 3 * 2 * 1 = 120`.

```javascript
function factorial(n) {
  // 1. Базовый случай (стоп-кран)
  if (n <= 1) return 1;

  // 2. Рекурсивный шаг
  // Чтобы посчитать 5!, нужно 5 умножить на результат 4!
  return n * factorial(n - 1);
}
```

## 🧬 Рекурсия в задаче Каррирования

В нашей задаче каррирования мы используем рекурсию, чтобы "наслаивать" аргументы.

```typescript
function curried(...args) {
  if (args.length >= func.length) {
    return func(...args); // БАЗОВЫЙ СЛУЧАЙ: аргументов хватило, выходим.
  }

  return (nextArg) => {
    // РЕКУРСИВНЫЙ ШАГ: вызываем curried снова, 
    // добавив новый аргумент к списку.
    return curried(...args, nextArg);
  };
}
```

## ⚠️ Главные опасности

### 1. Переполнение стека (Stack Overflow)
Если забыть базовый случай или сделать слишком много вложенных вызовов (обычно больше 10 000), память браузера закончится, и он выдаст ошибку `Maximum call stack size exceeded`.

### 2. Производительность
Иногда рекурсия работает медленнее, чем обычный цикл `for` или `while`, потому что на каждый вызов функции тратится немного времени и памяти.

## 📐 Шпаргалка для собеседования

*   **Что такое рекурсия?** Процесс, при котором функция вызывает саму себя.
*   **Из чего состоит?** Из базового случая (условие выхода) и рекурсивного шага.
*   **Где применяется?** В задачах на обход деревьев (DOM-дерево), в алгоритмах сортировки (QuickSort) и в функциональных техниках типа каррирования.
