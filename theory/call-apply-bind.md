# call, apply и bind: В чем разница?

Эти три метода используются для управления контекстом `this` в JavaScript. Главное различие между ними заключается в том, **когда** происходит вызов функции.

---

## 🚀 1. `call` и `apply` — Немедленный вызов

Эти методы заставляют функцию выполниться **прямо сейчас** с указанным контекстом.

*   **`call`**: принимает аргументы через запятую.
*   **`apply`**: принимает аргументы в виде массива.

```javascript
function greet(greeting, punctuation) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}

const person = { name: 'Alibek' };

greet.call(person, 'Hello', '!');  // Выполнится СРАЗУ
greet.apply(person, ['Hi', '.']); // Выполнится СРАЗУ
```

**Суть:** Вы заказываете еду в ресторане и едите её немедленно. Результатом вызова будет то, что возвращает сама функция.

---

## 📦 2. `bind` — Создание новой функции

Метод `bind` **НЕ вызывает** функцию сразу. Вместо этого он создает и возвращает **копию** функции с уже «приклеенным» контекстом и (опционально) аргументами.

```javascript
const person = { name: 'Alibek' };

const greetAlibek = greet.bind(person, 'Hello'); 
// Функция НЕ выполнилась. Мы просто создали "предзаказ".

// Вызываем когда захотим:
greetAlibek('!'); // Выведет: Hello, Alibek!
```

**Суть:** Это «предзаказ» или доставка на вечер. Вы подготавливаете функцию сейчас, чтобы вызвать её когда-нибудь потом.

---

## 🍽️ Аналогия: Ресторан vs Доставка

| Метод | Аналогия | Результат |
| :--- | :--- | :--- |
| **`call` / `apply`** | Вы пришли в ресторан, заказали и **сразу поели**. | Результат выполнения функции. |
| **`bind`** | Вы оформили **предзаказ** на вечер. Вам дали чек (функцию), по которому вы получите еду позже. | Новая функция-обертка. |

---

## 📐 Сводная таблица

| Метод | Вызывает функцию сразу? | Как принимает аргументы? | Что возвращает? |
| :--- | :--- | :--- | :--- |
| **`call`** | **Да** | Через запятую | Результат функции |
| **`apply`** | **Да** | Массивом | Результат функции |
| **`bind`** | **Нет** | Через запятую | **Новую функцию** |

---

## 💡 Почему это важно для каррирования?

В продвинутых реализациях каррирования используется именно `bind`, потому что он позволяет бесконечно создавать «копии» функции, накапливая в них аргументы, пока их не станет достаточно для финального выполнения.
