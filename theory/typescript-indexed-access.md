# TypeScript: Indexed Access Types (Доступ по индексу)

В TypeScript часто возникает задача получить тип элемента массива или значение свойства объекта, не дублируя код. Для
этого используются **Indexed Access Types**.

---

## 1. Получение типа элемента массива: `[number]`

Это самый частый паттерн, когда у вас есть массив данных (например, конфиг или моковые данные), и вы хотите создать тип
для одного элемента этого массива.

### Синтаксис:

```typescript
type ElementType = (typeof arrayName)[number]
```

### Как это работает:

1. **`typeof arrayName`**: TypeScript берет переменную и превращает её в тип (например, массив объектов).
2. **`[number]`**: Говорит TypeScript: «дай мне тип того, что находится под любым числовым индексом в этом массиве».

### Пример:

```typescript
const users = [
  { id: 1, name: 'Alibek', role: 'admin' },
  { id: 2, name: 'Ivan', role: 'user' },
]

// Автоматически создаст тип: { id: number; name: string; role: string; }
type User = (typeof users)[number]

// Теперь мы можем использовать User для типизации функций
function greetUser(user: User) {
  console.log(`Hello, ${user.name}!`)
}
```

## 2. Получение типа свойства объекта

Вы также можете получить тип конкретного поля объекта, используя строку в качестве индекса.

### Пример:

```typescript
interface AppConfig {
  api: {
    baseUrl: string
    timeout: number
  }
  theme: 'dark' | 'light'
}

// Получаем тип только для api
type ApiConfig = AppConfig['api']

// Получаем тип baseUrl внутри api
type BaseUrlType = AppConfig['api']['baseUrl'] // string
```

## 3. Комбинация: `as const` + Indexed Access

Это «продвинутый» уровень, который чаще всего используется в реальных проектах (например, для создания типов из конфигураций колонок в таблицах).

### Проблема «расширения» типов (Type Widening)
Обычно TypeScript видит `key: 'id'` как `key: string`. Чтобы TypeScript запомнил **конкретные значения**, нужно использовать `as const`.

### Пример: Извлечение литеральных типов из массива объектов

```typescript
const columns = [
  { label: 'ID', key: 'id' },
  { label: 'Name', key: 'name' },
  { label: 'Age', key: 'age' }
] as const; // Важно! Без этого key будет просто string

// 1. Сначала получаем тип одного элемента массива
// (typeof columns)[number] => { readonly label: "ID"; readonly key: "id"; } | ...

// 2. Затем обращаемся к полю 'key'
type SortField = (typeof columns)[number]['key']; 

// Результат: type SortField = "id" | "name" | "age"
```

### Как это работает по шагам:
1.  **`as const`**: Говорит TS: «этот массив и все его объекты — неизменяемые константы, сохрани их точные значения как типы».
2.  **`[number]`**: Разворачивает массив в объединение (Union) всех его объектов.
3.  **`['key']`**: Из каждого объекта в этом объединении забирает тип значения, лежащего в поле `key`.

---

## 📋 Почему это круто?

| Преимущество                    | Описание                                                              |
| :------------------------------ | :-------------------------------------------------------------------- |
| **DRY (Don't Repeat Yourself)** | Типы автоматически обновляются при изменении данных.                  |
| **Single Source of Truth**      | Массив данных сам определяет свой тип.                                |
| **Сокращение кода**             | Не нужно вручную описывать длинные интерфейсы для константных данных. |
| **Строгость**                   | Использование `as const` гарантирует, что вы не опечатаетесь в ключах. |

---

## 💡 Когда использовать?

1. Когда вы пишете **React-компоненты**, принимающие пропсом элемент из заранее определенного массива.
2. При работе с **константами** (например, список ролей, статусов или настроек).
3. При создании **generic-функций**, которые должны работать с элементами конкретных массивов.
4. Для создания **типов-ключей** на основе конфигурационных массивов (как в примере с `columns`).
