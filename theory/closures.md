# Замыкания (Closures)

**Замыкание** — это комбинация функции и лексического окружения, в котором эта функция была определена. Простыми словами: функция "запоминает" область видимости, в которой она была создана, даже если она вызывается вне этой области.

---

## 🏗️ 1. Как это работает в Каррировании?

В нашей задаче каррирования замыкание — это фундамент. Без него мы бы не смогли "накапливать" аргументы.

```typescript
function curry(func) {
  // func находится во внешней области видимости для curried
  return function curried(...args) {
    if (args.length >= func.length) {
      return func(...args);
    }

    // Эта стрелочная функция — ЗАМЫКАНИЕ. 
    // Она "захватывает" массив args и саму функцию curried.
    return (nextArg) => {
      // Когда мы вызываем эту функцию спустя время, она все еще 
      // имеет доступ к тем args, которые были в момент её создания.
      return curried(...args, nextArg);
    };
  };
}
```

## 🌟 2. Другие классические примеры

### А. Инкапсуляция и приватный счетчик
Замыкания позволяют создавать "приватные" переменные, к которым нет доступа снаружи.

```javascript
function createCounter() {
  let count = 0; // Эта переменная скрыта от внешнего мира

  return {
    increment() { count++; return count; },
    decrement() { count--; return count; },
    getCount() { return count; }
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.count); // undefined (переменная count защищена)
```

### Б. Фабрика функций (Конфигуратор)
Мы можем создавать специализированные функции на основе шаблона.

```javascript
function createMultiplier(factor) {
  // factor "замыкается" внутри возвращаемой функции
  return function(number) {
    return number * factor;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15
```

### В. Частичное применение (Partial Application)
Похоже на каррирование, но мы фиксируем часть аргументов сразу.

```javascript
function log(level, message) {
  console.log(`[${level}] ${message}`);
}

function createLogger(level) {
  return (message) => log(level, message);
}

const errorLog = createLogger('ERROR');
errorLog('Что-то пошло не так!'); // [ERROR] Что-то пошло не так!
```

## ⚖️ Плюсы и минусы

| Плюсы | Минусы |
| :--- | :--- |
| **Приватность:** Данные защищены от изменений извне. | **Память:** Переменные в замыкании не удаляются сборщиком мусора, пока жива ссылка на функцию. |
| **Чистота:** Функции остаются независимыми и предсказуемыми. | **Сложность отладки:** Иногда трудно отследить, какое именно значение "заперто" внутри. |
| **Гибкость:** Позволяет создавать динамические "фабрики" кода. | |

---

## 📐 Шпаргалка для собеседования

1.  **Что такое замыкание?** Это функция вместе со всеми внешними переменными, которые ей доступны.
2.  **Зачем они нужны?** Для имитации приватных методов, создания фабрик функций и сохранения состояния между вызовами.
3.  **Есть ли риски?** Да, риск утечек памяти, если создавать слишком много замыканий и не давать им удаляться.
