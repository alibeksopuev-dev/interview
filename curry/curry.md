# Разбор алгоритма Каррирования (`Currying`)

> Каррирование — это техника в функциональном программировании, которая превращает функцию с множеством аргументов в серию функций, каждая из которых принимает по одному аргументу.

---

## 📌 Введение: в чём идея?

Каррирование в основном встречается как задача на собеседованиях, но это отличный способ проверить понимание **замыканий (closures)**, **арности (arity)** и **проброса контекста (this forwarding)**.

### Основные термины:
1.  **Арность (Arity):** Количество аргументов, которые принимает функция (`func.length`).
2.  **Замыкание (Closure):** Комбинация функции и её лексического окружения (состояния), в котором она была объявлена.

У каррированной функции есть две фазы:
- Если аргументов **достаточно** — вызвать оригинальную функцию и вернуть результат.
- Если **нет** — вернуть новую функцию, которая "помнит" уже полученные аргументы и ждет следующие.

---

## ⏱️ 1. Временная сложность (Time Complexity): `O(1)` (для создания) / `O(N)` (для вызова)

- **Создание обертки:** `O(1)`, так как мы просто возвращаем новую функцию.
- **Вызов всей цепочки:** Если функция принимает `N` аргументов, нам потребуется `N` вызовов. Каждый вызов создает новое замыкание или копирует массив аргументов. Суммарно это `O(N)`.

---

## 💾 2. Пространственная сложность (Space Complexity): `O(N)`

Здесь `N` — количество аргументов (арность) исходной функции.
Каждый промежуточный вызов создает новую функцию и хранит массив уже переданных аргументов в памяти (в замыкании). В худшем случае мы храним `N` аргументов.

---

## 🔬 3. Пошаговый разбор (Трассировка) и "Накопление аргументов"

### Что значит "накопил аргументы"?
Фраза «накопил аргументы» означает, что функция «запоминает» те значения, которые вы ей уже передали, пока ждет остальные.
Обычная функция получает все аргументы сразу: `sum(1, 2, 3)`. При каррировании мы передаём аргументы по одному: `sum(1)(2)(3)`. 
Чтобы это работало, программа должна где-то хранить первую единицу, пока мы не передадим двойку, а затем хранить единицу и двойку, пока мы не передадим тройку. Это «хранилище» реализуется с помощью массива `args` и механизма **замыканий (closures)**.

Можно представить каррированную функцию как **корзину**, в которую мы по одному складываем продукты (аргументы), пока она не заполнится.

### Входные данные:
```typescript
function sum(a, b, c) { return a + b + c; } // arity = 3
const curriedSum = curry(sum);
```

### 🔁 Шаг 1: Вызов `const step1 = curriedSum(1)`
- **Что происходит внутри:** Вызов функции с аргументом `1`. 
- **Накопленные аргументы (`args`):** `[1]`
- **Проверка:** `args.length (1) >= func.length (3)`? **Нет.**
- **Результат:** Функция видит, что нужен еще аргумент. Запоминает единицу в замыкании (в массиве `args`) и возвращает новую функцию, которая ждет следующий шаг.

### 🔁 Шаг 2: Вызов `const step2 = step1(2)`
- **Что происходит внутри:** Вызов новой функции с аргументом `2`. Программа берёт старый массив `[1]` и к нему добавляет `2` (благодаря логике `[...args, arg]`).
- **Накопленные аргументы (`args`):** `[1, 2]`
- **Проверка:** `args.length (2) >= func.length (3)`? **Нет.**
- **Результат:** Снова запоминается текущее состояние, возвращается очередная функция-замыкание.

### 🔁 Шаг 3: Вызов `const result = step2(3)`
- **Что происходит внутри:** Передаём последний аргумент `3`.
- **Накопленные аргументы (`args`):** `[1, 2, 3]`
- **Проверка:** `args.length (3) >= func.length (3)`? **Да. Корзина полна!**
- **Результат:** Магия каррирования заканчивается. Вызывается оригинальная функция со всеми собранными аргументами: `func.apply(this, [1, 2, 3])`, и возвращается финальный результат `6`.

## 💡 4. Разбор реализации и FAQ

### Почему важен проброс `this`?
Если оригинальная функция использует `this` (например, метод объекта), то каррированная обертка должна гарантировать, что контекст не потеряется при финальном вызове. Именно поэтому используются методы `.apply(this, args)` или `.call(this, ...args)`.

### Зачем нужны стрелочные функции во внутренней реализации?
Внутренняя функция должна быть стрелочной, чтобы сохранить **лексический `this`** внешней обертки. Если использовать обычное `function`, то `this` внутри нее будет указывать на глобальный объект или `undefined` (в strict mode), если только мы не будем отслеживать его вручную.

### Вариант с `bind()`
```typescript
function curry(func: Function): Function {
  return function curried(this: any, ...args: any[]) {
    if (args.length >= func.length) {
      return func.apply(this, args);
    }
    // bind() автоматически создает замыкание с текущим контекстом и аргументами
    return curried.bind(this, ...args);
  };
}
```
Этот вариант считается более современным и гибким по нескольким причинам:

1.  **Гибкость вызовов (Variadic):** В отличие от первой реализации, `bind` позволяет передавать любое количество аргументов за один раз (например, `curried(1, 2)(3)` или `curried(1)(2, 3)`). Это происходит потому, что `bind(this, 1, 2)` создает новую функцию, в которую уже "вшиты" аргументы `1` и `2`. При следующем вызове новые значения просто добавятся в конец списка аргументов.
2.  **Автоматическое сохранение контекста:** `bind` сразу привязывает текущий `this`, поэтому нам не нужно заботиться о лексическом окружении или использовать стрелочные функции для проброса контекста.
3.  **Лаконичность:** Мы используем встроенный механизм языка для управления "корзиной" аргументов, что делает код чище.

## ⚠️ 5. Edge Cases (Граничные случаи)

1.  **Вызов без аргументов:** `curried()`
    В реализации выше пустой вызов не увеличивает количество накопленных аргументов, но сохраняет цепочку. Это полезно для контроля момента выполнения.
2.  **Функции с переменным числом аргументов (Rest parameters):**
    Если функция объявлена как `(...args) => ...`, то её `length` будет равен `0`. В этом случае каррирование через `func.length` сработает некорректно (выполнится сразу). Каррирование обычно применяется к функциям с фиксированным количеством аргументов.
3.  **Передача `undefined`:**
    Если передать `undefined` в стрелочную функцию (Вариант 1), она трактует это как "пустой вызов" и просто возвращает текущую стадию.
