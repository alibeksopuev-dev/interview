# Анализ функции `firstUniqueCharIndex`

## Задача

> Дана строка. Найти **индекс первого неповторяющегося символа**.  
> Если такого символа нет — вернуть `-1`.

Это классическая задача с LeetCode ([задача №387](https://leetcode.com/problems/first-unique-character-in-a-string/)).

---

## Сигнатура функции

```typescript
const firstUniqueCharIndex = (str: string): number => { ... }
```

| Параметр | Тип      | Описание                      |
|----------|----------|-------------------------------|
| `str`    | `string` | Входная строка для проверки   |

**Возвращает:** `number` — индекс первого уникального символа, или `-1`.

---

## Как работает алгоритм

Алгоритм состоит из **двух проходов по строке**.

### Проход 1 — Подсчёт частоты символов

```typescript
const charFrequencyMap = new Map<string, number>();

for (const char of str) {
  const currentCount = charFrequencyMap.get(char) ?? 0;
  charFrequencyMap.set(char, currentCount + 1);
}
```

Используется `Map<string, number>` — словарь, где:
- **ключ** — символ строки
- **значение** — сколько раз он встречается

После прохода для строки `"leetcode"`:

| Символ | Количество |
|--------|-----------|
| `l`    | 1         |
| `e`    | 3         |
| `t`    | 1         |
| `c`    | 1         |
| `o`    | 1         |
| `d`    | 1         |

---

### Проход 2 — Поиск первого уникального

```typescript
for (let charIndex = 0; charIndex < str.length; charIndex++) {
  const char = str[charIndex];

  if (charFrequencyMap.get(char) === 1) {
    return charIndex;
  }
}

return -1;
```

Перебираем строку **в оригинальном порядке** и возвращаем индекс первого символа, чья частота равна `1`.

> ⚠️ Важно: второй проход идёт по **строке**, а не по Map — это гарантирует, что мы найдём именно **первый** уникальный по порядку символа в тексте, а не по порядку вставки в Map.

## Детальная трассировка (Trace) по итерациям

Рассмотрим, как меняется состояние локальных переменных (`charFrequencyMap`, `charIndex`, `char`, `currentCount`) на каждом шаге алгоритма на реальном примере для строки `"leetcode"`.

### Проход 1: Заполнение частотного словаря (цикл `for...of`)

Построчно проходим по каждому символу строки `str`. На каждой итерации извлекаем текущее значение (`currentCount`) и записываем обновленное.

| Итерация | Символ (`char`) | `currentCount` (`get`) | Новое значение | Состояние `charFrequencyMap` (после шага) |
| :---: | :---: | :---: | :---: | :--- |
| **1** | `'l'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1 }` |
| **2** | `'e'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1, 'e' => 1 }` |
| **3** | `'e'` | `1` | `1 + 1 = 2` | `Map { 'l' => 1, 'e' => 2 }` |
| **4** | `'t'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1, 'e' => 2, 't' => 1 }` |
| **5** | `'c'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1, 'e' => 2, 't' => 1, 'c' => 1 }` |
| **6** | `'o'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1, 'e' => 2, 't' => 1, 'c' => 1, 'o' => 1 }` |
| **7** | `'d'` | `0` | `0 + 1 = 1` | `Map { 'l' => 1, 'e' => 2, 't' => 1, 'c' => 1, 'o' => 1, 'd' => 1 }` |
| **8** | `'e'` | `2` | `2 + 1 = 3` | `Map { 'l' => 1, 'e' => 3, 't' => 1, 'c' => 1, 'o' => 1, 'd' => 1 }` |

После полного первого прохода мы имеем заполненный словарь с частотами всех символов.

---

### Проход 2: Поиск первого уникального символа (цикл `for`)

Теперь идём по индексам от `0` до `str.length - 1` (для `"leetcode"` длина 8).

| Итерация | `charIndex` | `char` (`str[i]`) | Извлечение `get(char)` | Проверка `=== 1` | Действие функции |
| :---: | :---: | :---: | :---: | :---: | :--- |
| **1** | `0` | `'l'` | `1` | `true` | `return 0;` (Возвращаем индекс 0) |

Функция завершает работу и **сразу возвращает 0** на первой же итерации второго цикла, так как символ `'l'` оказался уникальным.

> **А если разобраться на примере `"loveleet"`?**
> Во втором проходе стек итераций выглядел бы так:
> * **Итерация 1:** `charIndex = 0` → `'l'` → `get('l') == 2` → `false` (продолжаем)
> * **Итерация 2:** `charIndex = 1` → `'o'` → `get('o') == 1` → `true` → `return 1;`
> 
> *Итог: Возвращается индекс 1, так как 'o' первый символ, чья частота равна 1.*

## Дополнительный пример без уникальных символов

### Пример: `"aabb"` → `-1`

```text
Частоты после 1-го прохода: Map { a=2, b=2 }

Проход 2 (Стек итераций):
  i=0 → 'a' → count=2 → ❌ продолжаем
  i=1 → 'a' → count=2 → ❌ продолжаем
  i=2 → 'b' → count=2 → ❌ продолжаем
  i=3 → 'b' → count=2 → ❌ продолжаем
```
Функция завершает цикл `for`, так и не наткнувшись на `return`, и доходит до конца тела функции, где выполняется `return -1;`.

## Сложность алгоритма

| Характеристика    | Значение   | Объяснение                                             |
|-------------------|------------|--------------------------------------------------------|
| **Время O(n)**    | Линейное   | Два линейных прохода по строке длиной `n`              |
| **Память O(k)**   | Константная| Map хранит не более `k` уникальных символов (алфавит) |

> Если алфавит фиксирован (например, только строчные латинские буквы), то `k ≤ 26` — память фактически **O(1)**.

---

## Почему `Map`, а не объект `{}`?

| Критерий              | `Map`                        | Обычный объект `{}`          |
|-----------------------|------------------------------|------------------------------|
| Порядок ключей        | ✅ Гарантирован по вставке   | ⚠️ Не всегда предсказуем     |
| Ключи любого типа     | ✅ Да                        | ❌ Только строки/символы     |
| Встроенный `.get()`   | ✅ Явный и типобезопасный    | ⚠️ Через `[]` без типов      |
| Производительность    | ✅ Оптимизирован для частых операций чтения/записи | ✅ Тоже быстр |

Использование `Map` — осознанный и правильный выбор для этой задачи.

---

## Оператор `??` (Nullish Coalescing)

```typescript
const currentCount = charFrequencyMap.get(char) ?? 0;
```

`??` возвращает правый операнд, если левый равен `null` или `undefined`.  
Здесь: если символ не найден в Map (`.get()` вернёт `undefined`), используем `0` как начальное значение счётчика.

## Возможные улучшения

### Вариант с `for...of` для второго прохода

```typescript
for (const [charIndex, char] of [...str].entries()) {
  if (charFrequencyMap.get(char) === 1) return charIndex;
}
```

Более декларативный стиль, но чуть менее производительный из-за создания массива.

### Вариант через `Array.findIndex`

```typescript
return [...str].findIndex(char => charFrequencyMap.get(char) === 1);
```

Однострочный, читаемый, возвращает `-1` автоматически если не найдено — идеально для интервью.

## Итог

Функция `firstUniqueCharIndex` реализует **оптимальный алгоритм O(n)** для поиска первого неповторяющегося символа в строке:

1. 🗺️ **Строим Map** частот за один проход
2. 🔍 **Ищем** первый символ с частотой `1` за второй проход
3. 📤 **Возвращаем** его индекс или `-1`

Код хорошо читаем, грамотно прокомментирован и использует современные возможности TypeScript и ES6+.
