# Разбор алгоритма Вставки интервала (insertInterval) и Big O

---

## 📌 Описание задачи и алгоритма

Дан **отсортированный** массив непересекающихся интервалов. Нужно вставить новый интервал `[start, end]` так, чтобы массив остался отсортированным и без пересечений — при необходимости слить накладывающиеся отрезки.

Отличие от `mergeIntervals`: входной массив уже отсортирован и без пересечений. Это позволяет решить задачу за **`O(n)`** без предварительной сортировки.

---

## 🔧 Два подхода

### Подход 1: Линейное сканирование (три фазы)

**Ключевая идея:** Разбить обход массива на три чётко разделённые фазы:
1. Интервалы **до** нового (нет пересечений) → просто копируем
2. Интервалы, которые **пересекаются** с новым → сливаем в один
3. Интервалы **после** нового (нет пересечений) → просто копируем

Поскольку массив отсортирован, каждый индекс `i` посещается ровно один раз.

### Подход 2: Бинарный поиск + стандартное слияние

**Ключевая идея:** Найти позицию вставки за `O(log n)` с помощью бинарного поиска по началу интервала, вставить туда `newInterval`, а затем запустить стандартный алгоритм `mergeIntervals`.

---

## ⏱️ 1. Временная сложность (Time Complexity)

### Подход 1 — Линейное сканирование: `O(n)`

| Шаг | Операция | Сложность |
|-----|----------|-----------|
| Фаза 1 | `while` — добавить "левые" интервалы | `O(k₁)` |
| Фаза 2 | `while` — слить пересекающиеся | `O(k₂)` |
| Фаза 3 | `while` — добавить "правые" интервалы | `O(k₃)` |
| **Итого** | `k₁ + k₂ + k₃ = n` | **`O(n)`** |

Каждый из `n` элементов посещается ровно один раз — это линейное время.

### Подход 2 — Бинарный поиск: `O(n)`

| Шаг | Операция | Сложность |
|-----|----------|-----------|
| Бинарный поиск | Найти позицию вставки | `O(log n)` |
| `splice` | Вставить в середину массива | `O(n)` — сдвиг элементов |
| Цикл слияния | Пройтись по массиву длиной `n + 1` | `O(n)` |
| **Итого** | `O(log n) + O(n) + O(n)` | **`O(n)`** |

Несмотря на использование бинарного поиска, узкое место — `splice` (сдвиг элементов). Итог тот же: `O(n)`.

> **Вывод:** Оба решения дают `O(n)`. Подход 1 проще и чуть быстрее на практике (нет `splice`). Подход 2 демонстрирует переиспользование паттерна из `mergeIntervals`.

---

## 💾 2. Пространственная сложность (Space Complexity): `O(n)`

- В обоих подходах создаётся массив результатов `res`.
- **Худший сценарий** — нет ни одного пересечения: `res` содержит все `n` исходных интервалов плюс `newInterval`. Это `O(n)`.
- В подходе 2 дополнительно изменяется входной массив `intervals` через `splice` (in-place расширение), что тоже `O(n)`.

**Итог по памяти:** **`O(n)`** для обоих решений.

---

## 🔍 3. Пошаговая трассировка — Подход 1

### Пример 1: `intervals = [[1,2],[3,5],[6,10],[11,12],[13,16]]`, `newInterval = [8,15]`

**Начало:** `i = 0`, `res = []`, `newInterval = [8, 15]`

---

**── Фаза 1: интервалы до нового ──**

> **i = 0. Текущий: `[1, 2]`**
> - Условие: `2 < 8`? ✅ Да
> - Действие: добавляем в `res`
> - `res = [[1, 2]]`, `i = 1`

> **i = 1. Текущий: `[3, 5]`**
> - Условие: `5 < 8`? ✅ Да
> - Действие: добавляем в `res`
> - `res = [[1, 2], [3, 5]]`, `i = 2`

> **i = 2. Текущий: `[6, 10]`**
> - Условие: `10 < 8`? ❌ Нет → выходим из Фазы 1

---

**── Фаза 2: слияние пересекающихся ──**

> **i = 2. Текущий: `[6, 10]`**
> - Условие: `15 >= 6`? ✅ Да — есть пересечение
> - `newInterval[0] = min(8, 6) = 6`
> - `newInterval[1] = max(15, 10) = 15`
> - `newInterval = [6, 15]`, `i = 3`

> **i = 3. Текущий: `[11, 12]`**
> - Условие: `15 >= 11`? ✅ Да — есть пересечение
> - `newInterval[0] = min(6, 11) = 6`
> - `newInterval[1] = max(15, 12) = 15`
> - `newInterval = [6, 15]`, `i = 4`

> **i = 4. Текущий: `[13, 16]`**
> - Условие: `15 >= 13`? ✅ Да — есть пересечение
> - `newInterval[0] = min(6, 13) = 6`
> - `newInterval[1] = max(15, 16) = 16`
> - `newInterval = [6, 16]`, `i = 5`

> **i = 5. Выход из Фазы 2** (`i >= n`)
> - Добавляем слитый интервал: `res.push([6, 16])`
> - `res = [[1, 2], [3, 5], [6, 16]]`

---

**── Фаза 3: интервалы после нового ──**

> `i = 5 = n` → массив исчерпан, Фаза 3 пуста

✅ **Итог:** `[[1, 2], [3, 5], [6, 16]]`

---

### Пример 2: `intervals = [[1,2],[3,4],[5,6],[7,8]]`, `newInterval = [2,5]`

**Начало:** `i = 0`, `res = []`, `newInterval = [2, 5]`

**── Фаза 1 ──**

> **i = 0. `[1, 2]`**: `2 < 2`? ❌ → Фаза 1 пуста (первый же интервал пересекается)

**── Фаза 2 ──**

> **i = 0. `[1, 2]`**: `5 >= 1`? ✅
> - `newInterval = [min(2,1), max(5,2)] = [1, 5]`, `i = 1`

> **i = 1. `[3, 4]`**: `5 >= 3`? ✅
> - `newInterval = [min(1,3), max(5,4)] = [1, 5]`, `i = 2`

> **i = 2. `[5, 6]`**: `5 >= 5`? ✅
> - `newInterval = [min(1,5), max(5,6)] = [1, 6]`, `i = 3`

> **i = 3. `[7, 8]`**: `6 >= 7`? ❌ → выходим из Фазы 2
> - `res.push([1, 6])` → `res = [[1, 6]]`

**── Фаза 3 ──**

> **i = 3. `[7, 8]`**: добавляем → `res = [[1, 6], [7, 8]]`

✅ **Итог:** `[[1, 6], [7, 8]]`

---

### Пример 3: `intervals = [[2,4],[6,8],[10,12]]`, `newInterval = [15,17]`

**── Фаза 1 ──**

> `[2,4]`: `4 < 15`? ✅ → `res = [[2,4]]`  
> `[6,8]`: `8 < 15`? ✅ → `res = [[2,4],[6,8]]`  
> `[10,12]`: `12 < 15`? ✅ → `res = [[2,4],[6,8],[10,12]]`

**── Фаза 2 ──** (`i = 3 = n`) — пуста

> `res.push([15, 17])` → `res = [[2,4],[6,8],[10,12],[15,17]]`

**── Фаза 3 ──** — пуста

✅ **Итог:** `[[2, 4], [6, 8], [10, 12], [15, 17]]`

---

## 📊 Сравнение с mergeIntervals

| | `mergeIntervals` | `insertInterval` |
|---|---|---|
| Входной массив | Неотсортированный | Уже отсортированный и без пересечений |
| Нужна сортировка? | ✅ Да, `O(n log n)` | ❌ Нет |
| Time Complexity | `O(n log n)` | `O(n)` |
| Space Complexity | `O(n)` | `O(n)` |
| Паттерн | Сортировка + линейное сканирование | Линейное сканирование (3 фазы) |
