# Разбор алгоритма Максимального произведения подмассива (maxProductSubArray) и Big O

---

## 📌 Описание задачи и алгоритма
Задача: найти непрерывный фрагмент внутри массива (подмассив), перемножение чисел которого даст самый большой результат. 

**Ключевая идея (Префикс и Суффикс):**
Если бы все числа были положительными, мы бы просто перемножили весь массив. Но проблема в том, что:
1. **Отрицательные числа** могут все испортить (сделать плюс минусом), но два отрицательных числа дают плюс! Поэтому иногда выгодно захватить отрицательное число в надежде встретить еще одно дальше.
2. **Нули (`0`)** уничтожают все накопленное произведение безвозвратно. 

Алгоритм вычисляет произведение **одновременно с двух краев массива**: слева направо (префиксное) и справа налево (суффиксное).
Если встречается ноль, то в этой точке считается, что непрерывная цепочка порвалась. Накопленное произведение аппаратно сбрасывается, и отсчет снова начинается с текущего числа (как бы рождается новый подмассив). Сканирование сразу с двух концов покрывает все возможные нечетные/четные количества отрицательных чисел в блоках между нулями.

---

## ⏱️ 1. Временная сложность (Time Complexity): `O(n)`
- У нас есть только один цикл `for (let i = 0; i < n; i++)`, который проходит по каждому элементу массива ровно один раз.
- Внутри цикла выполняются простейшие арифметические операции: проверки на ноль `===`, умножение `*` и поиск максимума `Math.max()`. Все эти действия занимают время **`O(1)`**.

**Итог:** Зависимость строго линейная. Временная сложность составляет **`O(n)`**.

---

## 💾 2. Пространственная сложность (Space Complexity): `O(1)`
- Нам не требуется выделять дополнительную память под массивы, стеки или объекты, независимо от длины входящих данных `arrayLength`.
- Алгоритм обходится всего тремя переменными (`prefixProduct`, `suffixProduct` и `maxProduct`), которые просто перезаписывают свои значения.

**Итог:** Константная (неизменная) сложность. Память **`O(1)`**.

---

## 🔍 3. Пошаговый разбор (Трассировка)

Возьмем пример из задачи с интересными переходами (там есть и 0, и отрицательные числа): 
`numbers = [1, 2, 0, -1, 8, -4]` (длина `arrayLength = 6`)

**Начальные состояния до старта:**
- `prefixProduct` = `1`
- `suffixProduct` = `1`
- `maxProduct` = `-Infinity` (самое маленькое, чтобы первый же шаг перебил рекорд)

*Идем по циклу от 0 до 5, двигаем указатели с обоих концов встречно:*

> **Шаг `i = 0`. Слева: `1` (`[0]`), Справа: `-4` (`[5]`)**
> - **`prefixProduct` (слева):** `1 * 1 = 1`
> - **`suffixProduct` (справа):** `1 * -4 = -4`
> - 🟢 **Рекорд (`maxProduct`):** Максимум из `(-Infinity, 1, -4)`. Новое лидерство: **`1`**.

> **Шаг `i = 1`. Слева: `2` (`[1]`), Справа: `8` (`[4]`)**
> - **`prefixProduct` (слева):** `1 * 2 = 2`
> - **`suffixProduct` (справа):** `-4 * 8 = -32`
> - 🟢 **Рекорд (`maxProduct`):** Максимум из `(1, 2, -32)`. Новое лидерство: **`2`**.

> **Шаг `i = 2`. Слева: `0` (`[2]`), Справа: `-1` (`[3]`)**
> - **`prefixProduct` (слева):** `2 * 0 = 0` *(встретили 0!)*
> - **`suffixProduct` (справа):** `-32 * -1 = 32` *(минус на минус дал огромный плюс)*
> - 🟢 **Рекорд (`maxProduct`):** Максимум из `(2, 0, 32)`. Новое лидерство: **`32`**.

> **Шаг `i = 3`. Слева: `-1` (`[3]`), Справа: `0` (`[2]`)**
> - ⚠️ **`prefixProduct` (слева):** Прошлое значение было `0`, **цепочка разорвана!** `prefixProduct` не умножается, а стартует заново с текущего числа: `-1`.
> - **`suffixProduct` (справа):** `32 * 0 = 0` *(теперь справа дошли до нуля!)*
> - 🔴 **Рекорд (`maxProduct`):** Максимум из `(32, -1, 0)`. Рекорд не побит, остается **`32`**.

> **Шаг `i = 4`. Слева: `8` (`[4]`), Справа: `2` (`[1]`)**
> - **`prefixProduct` (слева):** `-1 * 8 = -8`
> - ⚠️ **`suffixProduct` (справа):** Прошлое значение было `0`, **цепочка разорвана!** `suffixProduct` стартует заново с числа: `2`.
> - 🔴 **Рекорд (`maxProduct`):** Максимум из `(32, -8, 2)`. Рекорд остается **`32`**.

> **Шаг `i = 5`. Слева: `-4` (`[5]`), Справа: `1` (`[0]`)**
> - **`prefixProduct` (слева):** `-8 * -4 = 32` *(минус на минус снова дал жирный плюс)*
> - **`suffixProduct` (справа):** `2 * 1 = 2`
> - 🔴 **Рекорд (`maxProduct`):** Максимум из `(32, 32, 2)`. Рекорд **`32`**.

**🏁 Итог:** Цикл закончен. Возвращается максимальное найденное произведение: `32`.
Оно получилось благодаря "суффиксному" проходу (начиная с конца массива: `-4 * 8 * -1`), который игнорировал ноль, так как находился до него.
