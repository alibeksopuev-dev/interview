# Разбор алгоритма Слияния интервалов (mergeIntervals) и Big O

---

## 📌 Описание задачи и алгоритма

Задача алгоритма — взять список временных (или числовых) отрезков, найти все те, которые "накладываются" друг на друга, и объединить их в сплошные длинные отрезки.

Например: `[1, 4]` и `[2, 3]` пересекаются (второй лежит внутри первого), поэтому после объединения будет один большой отрезок `[1, 4]`.

**Ключевая идея:** 
Алгоритм начинает с **обязательной сортировки**. Если не отсортировать отрезки, нам пришлось бы сравнивать каждый элемент с каждым. 
Но если мы отсортируем отрезки по их **началу**, мы гарантируем, что каждый следующий отрезок начнётся либо позже, либо одновременно с предыдущим. Благодаря этому, нам достаточно один раз пройтись по отсортированному массиву и сравнивать каждый следующий отрезок только с "последним добавленным" (пересекается с ним или нет). Это классический подход с использованием паттерна "Линейное сканирование массива".

---

## ⏱️ 1. Временная сложность (Time Complexity): `O(n log n)`

Разложим алгоритм на шаги:

1. **Сортировка:** 
   ```typescript
   intervals.sort((a, b) => a[0] - b[0]);
   ```
   Встроенный метод сортировки JavaScript имеет сложность **`O(n log n)`**, где `n` — это количество отрезков.

2. **Один проход цикла `for...of`:**
   ```typescript
   for (const interval of intervals) { ... }
   ```
   Сортировка уже сделала за нас самую тяжелую работу. Теперь мы проходим по массиву ровно один раз. Получение конца интервала, функции `Math.max()`, операции присваивания и `push()` массива выполняются мгновенно за **`O(1)`**. Поэтому цикл отнимает ровно линейное время: **`O(n)`**.

**Итог:** Поскольку при миллионах элементов операция сортировки `O(n log n)` будет самой "медленной" частью, она перекрывает `O(n)`. Итоговое время выполнения оценивается как **`O(n log n)`**.

---

## 💾 2. Пространственная сложность (Space Complexity): `O(n)`

- Встроенный метод `.sort()`, в зависимости от движка (V8, JavaScriptCore), может потребовать под капотом от `O(log n)` до `O(n)` дополнительной памяти для рекурсивных вызовов сортировки (в случае Timsort — `O(n)`).
- Мы создаём массив ответов `result`. Если нам повезёт и все отрезки на вход подадут в виде одной сплошной линии склеивания (например `[1,2], [2,3], [3,4]`), в ответе будет лежать всего 1 отрезок (мало памяти). Но по правилам Big O мы рассматриваем **худший сценарий**. 
- Худший сценарий — когда в массиве нет ни одного пересечения: `[1, 2], [3, 4], [5, 6]`. В этом случае мы просто скопируем все `n` элементов в массив `result`.

**Итог по памяти:** Сложность по памяти будет составлять **`O(n)`**.

## 🔍 3. Пошаговый разбор (Трассировка)

Возьмём самый сложный пример из ваших комментариев (`array3`): 
Исходный массив: `[[11, 12], [2, 3], [5, 7], [1, 4], [8, 10], [6, 8]]`

**Шаг 0. Сортировка:**
После строчки `intervals.sort`, наш массив теперь выглядит так:
`[[1, 4], [2, 3], [5, 7], [6, 8], [8, 10], [11, 12]]`

Мы кладём в массив `result` самый первый элемент. Теперь начальное состояние:
`result = [[1, 4]]`

*Поехали по циклу (`for const interval of intervals`):*

> **Шаг 1. Текущий отрезок `=` `[1, 4]`**
> - **`recent`** (последний сохраненный): `[1, 4]`
> - 🟢 **Пересечение? Да** (`4 >= 1`)
> - **Действие:** Это первый элемент, совпадает с инициализацией. Конец = `Math.max(4, 4)` = `4`.
> - **В массиве `result`:** `[[1, 4]]`

> **Шаг 2. Текущий отрезок `=` `[2, 3]`**
> - **`recent`** (последний сохраненный): `[1, 4]`
> - 🟢 **Пересечение? Да** (`4 >= 2`)
> - **Действие:** Отрезок полностью помещается внутри. Сдвигаем массив в максимум: `Math.max(4, 3)` = `4`. Интервал `[1, 4]` полностью "проглотил" `[2, 3]`.
> - **В массиве `result`:** `[[1, 4]]`

> **Шаг 3. Текущий отрезок `=` `[5, 7]`**
> - **`recent`** (последний сохраненный): `[1, 4]`
> - 🔴 **Пересечение? Нет** (`4 < 5`)
> - **Действие:** Пересечения нет, появился разрыв. Интервал `[1, 4]` закрыт. Добавляем новый "островок" в массив откликов.
> - **В массиве `result`:** `[[1, 4], [5, 7]]`

> **Шаг 4. Текущий отрезок `=` `[6, 8]`**
> - **`recent`** (последний сохраненный): `[5, 7]`
> - 🟢 **Пересечение? Да** (`7 >= 6`)
> - **Действие:** Найдено пересечение! Конец сдвигается: `Math.max(7, 8)` = `8`.
> - **В массиве `result`:** `[[1, 4], [5, 8]]`

> **Шаг 5. Текущий отрезок `=` `[8, 10]`**
> - **`recent`** (последний сохраненный): `[5, 8]`
> - 🟢 **Пересечение? Да** (`8 >= 8`)
> - **Действие:** Отрезки соприкасаются. Сдвигаем край слияния: `Math.max(8, 10)` = `10`.
> - **В массиве `result`:** `[[1, 4], [5, 10]]`

> **Шаг 6. Текущий отрезок `=` `[11, 12]`**
> - **`recent`** (последний сохраненный): `[5, 10]`
> - 🔴 **Пересечение? Нет** (`10 < 11`)
> - **Действие:** Снова разрыв. Интервал `[5, 10]` больше не изменится. Мы кладем `[11, 12]` в массив отдельным ответом.
> - **В массиве `result`:** `[[1, 4], [5, 10], [11, 12]]`

Цикл завершен, функция возвращает `result`.
✅ **Итог:** `[[1, 4], [5, 10], [11, 12]]`
