# Разбор алгоритма `hasDuplicates` (Поиск дубликатов) и оценка сложности (Big O)

> Простыми словами: почему использование структуры данных `Set` — это лучший выбор и как мы получаем сложность `O(n)`.

---

## 📌 Введение в алгоритм
В основе этого решения лежит отличная структура данных — **`Set` (множество)**, которая великолепно подходит для отслеживания уникальных значений.
Суть проста: мы проходим по массиву `numbers` шаг за шагом. На каждом шаге мы задаем себе вопрос: видели ли мы уже это число (то есть, есть ли оно в нашем `Set`)?
- Если число уже найдено в `Set` — у нас дубликат! (Сразу возвращаем `true`)
- Если числа нет в `Set` — добавляем его туда и продолжаем цикл.

Здесь `n` — это длина массива `numbers`.

---

## ⏱️ 1. Временная сложность (Time Complexity): `O(n)`

Давай посчитаем количество "шагов", которые делает твоя программа при проверке:

```typescript
for (let i = 0; i < n; i++) {
  if (seen.has(numbers[i])) { ... }
  seen.add(numbers[i]);
}
```

* **Сколько раз сработает этот цикл (`for`)?**
  В самом худшем случае (когда все элементы уникальны и дубликатов нет), цикл пройдет по всем элементам от начала и до самого конца. Это ровно `n` витков.
* **Что происходит внутри цикла?**
  - Метод `seen.has()` выполняет проверку наличия элемента в `Set`. В отличие от массивов (где `[].includes()` занимает время `O(n)`), в структуре `Set` поиск реализован через хэширование и выполняется практически мгновенно — за константное время: **`O(1)`**.
  - Метод `seen.add()` тоже работает очень быстро — за **`O(1)`**.

### 🧮 Итог как математика:
Цикл делает в худшем случае `n` витков. На каждом микро-операции (вставка/проверка) происходят за идеальную сложность `O(1)`.
`n` умножить на 1 = **`O(n)`**.

> 🏆 **Твоё решение:** Алгоритм работает с максимально возможной скоростью для этой задачи. Быстрее `O(n)` сделать поиск в неотсортированном массиве невозможно в принципе — ведь чтобы убедиться в уникальности всего массива, нужно прочитать каждый его элемент хотя бы один раз.

---

## 💾 2. Пространственная сложность (Space Complexity): `O(n)`

**Пространственная сложность** показывает, сколько дополнительной "кратковременной" памяти (ОЗУ) потребуется алгоритму, когда размер массива начнет расти.

Давай посмотрим на память, которую ты выделяешь:
```typescript
const seen: Set<number> = new Set();
```

* Если на вход поступит массив из `10` уникальных чисел, к концу выполнения цикла в `Set` будет бережно сохранено `10` элементов.
* Если на вход поступит миллион уникальных чисел, `Set` раздуется и будет хранить `1,000,000` элементов в памяти.
* Зависимость очевидна и линейна: чем больше исходный массив (`n`), тем в конечном итоге больше чисел нам придется сохранить в `Set`.

### 🧮 Итог как математика:
Поскольку требуемая алгоритмом дополнительная перераспределяемая память **растет пропорционально** размеру массива `n`, пространственная сложность будет линейной: **`O(n)`**.

(*Примечание: Это классический компромисс в компьютерных науках (Time/Space tradeoff). Мы "заплатили" дополнительной памятью `O(n)`, чтобы кардинально сократить время работы алгоритма с `O(n^2)` до `O(n)`.*)

---

## 🥊 Сравнение с неэффективным решением (Два вложенных цикла)

Иногда новички пишут такой код, чтобы не выделять дополнительную память:

```javascript
function findDuplicates(numbers) {
  for (let i = 0; i < numbers.length; i++) {
    for (let j = i + 1; j < numbers.length; j++) { // Вложенный цикл!
      if (numbers[i] === numbers[j]) {
        return true;
      }
    }
  }
  return false;
}
```

Почему твой подход через `Set` выигрывает всухую?

### ⏱ Временная сложность (Скорость)
- **Твой код (Set):** `O(n)`. Делает 1 просмотр исходного массива.
- **Два цикла:** `O(n^2)`. Делает просмотр всего оставшегося массива *для каждого* элемента. Если мы передадим массив всего на 100,000 чисел, такой алгоритм сделает миллиарды шагов и "подвесит" браузер пользователя на несколько минут.

### 💾 Пространственная сложность (Память)
- **Твой код (Set):** `O(n)`. Занимает немного дополнительную память под хэш-таблицу.
- **Два цикла:** `O(1)`. Работает без дополнительной памяти.

> 🏆 **Итог:** Вариант с `Set` является признанным стандартом индустрии (Best Practice). В современном вебе намного выгоднее потратить крупицу дополнительной оперативной памяти, чем заставлять процессор пользователя греться, выполняя вложенные медленные циклы. Твой код написан отлично и по канонам эффективных алгоритмов.
