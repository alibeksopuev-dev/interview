# 🔢 Find Missing Number — Глубокий разбор задачи

> **Сложность:** Easy &nbsp;|&nbsp; **Язык:** TypeScript / JavaScript &nbsp;|&nbsp; **Время:** ~25 мин

---

## 📋 Условие задачи

Дан массив `numbers`, содержащий `n` **различных** чисел из диапазона `[0, n]`.  
Ровно одно число из этого диапазона **отсутствует**.

**Необходимо найти** это пропущенное число.

### Входные данные

| Параметр   | Тип        | Описание                                  |
|------------|------------|-------------------------------------------|
| `numbers`  | `number[]` | Массив из `n` уникальных целых чисел      |

**Возвращает:** `number` — единственное отсутствующее число из `[0, n]`.

### Ограничения

```
n == numbers.length
1 <= n <= 10^4
0 <= numbers[i] <= n
Все числа в массиве уникальны
```

### Пример 1 — пропущено среднее

```
Input:  numbers = [3, 0, 1]
Output: 2
```
> Диапазон [0, 3]: числа 0, 1, 3 есть → пропущено **2**.


## 🔴 Подход 1: Сортировка (Brute Force)

### Идея

Если **отсортировать** массив, каждый элемент должен стоять на позиции, равной своему значению: `numbers[i] === i`.  
Первая позиция, где это не выполняется — и есть пропущенное число.

### Алгоритм

1. Отсортировать массив по возрастанию
2. Пройти по отсортированному массиву
3. Если `numbers[i] !== i` → вернуть `i`
4. Если цикл завершился без нахождения — вернуть `n` (пропущен последний)

### Код

```typescript
export function findMissingNumberSorting(numbers: number[]): number {

  // Шаг 1: Сортировка по возрастанию
  // Пример: [3, 0, 1] → [0, 1, 3]
  numbers.sort((a, b) => a - b);

  // Шаг 2: Ищем первую позицию, где numbers[i] !== i
  for (let i = 0; i < numbers.length; i++) {
    if (numbers[i] !== i) {
      return i; // нашли пропуск — возвращаем индекс
    }
  }

  // Шаг 3: Все элементы на своих местах → пропущен n
  // Пример: [0, 1, 2] → возвращаем 3
  return numbers.length;
}
```

### Big-O анализ

| Сложность  | Обозначение    | Пояснение                                    |
|------------|----------------|----------------------------------------------|
| **Время**  | **O(n log n)** | Доминирует сортировка; линейный проход O(n)  |
| **Память** | **O(1)**       | Сортировка in-place, доп. структур нет       |

## 🟢 Подход 2: Математическое свойство суммы (Optimal)

### Идея

Сортировка, скорее всего, избыточна — пропущенное число можно найти,  
используя **математическое свойство суммы целых чисел от 0 до n**.

Сумма всех чисел от `0` до `n` заранее известна.  
Если вычесть из неё фактическую сумму элементов массива — получим пропущенное число.  
Это устраняет необходимость сортировки и снижает сложность до **O(n)**.

### Алгоритм

1. Инициализировать `expectedSum = 0`
2. Вычислить ожидаемую сумму чисел от `0` до `n` циклом, записать в `expectedSum`
3. Инициализировать `actualSum = 0`
4. Пройти по массиву и вычислить сумму всех его элементов, записать в `actualSum`
5. Вычесть `actualSum` из `expectedSum` — результат и есть пропущенное число
6. Вернуть результат

### Код

```typescript
export default function findMissingNumberInSequence(numbers: number[]): number {
  const n = numbers.length;

  // Шаг 1–2: Ожидаемая сумма чисел от 0 до n
  // Пример (n=3): 0+1+2+3 = 6
  let expectedSum = 0;
  for (let i = 0; i <= n; i++) {
    expectedSum += i;
  }

  // Шаг 3–4: Фактическая сумма элементов массива
  // Пример [3,0,1]: 3+0+1 = 4
  let actualSum = 0;
  for (const num of numbers) {
    actualSum += num;
  }

  // Шаг 5–6: Пропущенное число = разность сумм
  // 6 - 4 = 2 ✓
  return expectedSum - actualSum;
}
```

### Big-O анализ

| Сложность  | Обозначение | Пояснение                                               |
|------------|-------------|---------------------------------------------------------|
| **Время**  | **O(n)**    | Два линейных прохода: вычисление обеих сумм             |
| **Память** | **O(1)**    | Используем только переменные-счётчики, доп. память не нужна |

## ✨ Бонус: Формула Гаусса (истинно O(1) по времени для суммы)

### Идея

Вместо цикла для `expectedSum` применяем **формулу Гаусса**:

```
Сумма(0..n) = n * (n + 1) / 2
```

Это убирает первый проход, оставляя только один — для `actualSum`.

### Код

```typescript
export function findMissingNumberGauss(numbers: number[]): number {
  const n = numbers.length;

  // Формула Гаусса: O(1)
  const expectedSum = (n * (n + 1)) / 2;

  // Фактическая сумма: O(n)
  const actualSum = numbers.reduce((sum, num) => sum + num, 0);

  return expectedSum - actualSum;
}
```

### Big-O анализ

| Сложность  | Обозначение | Пояснение                                              |
|------------|-------------|--------------------------------------------------------|
| **Время**  | **O(n)**    | Только один проход (для actualSum); expectedSum — O(1) |
| **Память** | **O(1)**    | Только две числовые переменные                         |

## ⚖️ Сравнение алгоритмов

| Подход                  | Время          | Память | Когда использовать                     |
|-------------------------|----------------|--------|----------------------------------------|
| **Сортировка**          | O(n log n)     | O(1)   | Простота реализации, малые данные      |
| **Сумма (цикл)**        | O(n)           | O(1)   | Наглядность логики                     |
| **Формула Гаусса**      | O(n)           | O(1)   | Production, оптимальное решение        |

### Почему математика лучше сортировки?

При `n = 10 000`:

```
Сортировка:  n * log₂(n) ≈ 132 877 операций
Математика:  n            = 10 000 операций
```

> Разница в **~13 раз** при `n = 10 000` и растёт по мере увеличения `n`.

## 🔑 Ключевые выводы

> [!NOTE]
> **Почему `return numbers.length`?** Если все элементы `numbers[i] === i` — значит, присутствуют числа от `0` до `n-1`, а пропущено `n`. Это крайний случай, который нельзя упустить ни при одном из подходов.

> [!TIP]
> **Формула Гаусса** `n*(n+1)/2` — классический трюк для задач с суммами последовательностей. Запомните его: она встречается во многих задачах на массивы и числа.

> [!IMPORTANT]
> **Сортировка мутирует массив:** `numbers.sort(...)` изменяет исходный массив in-place. Если необходимо сохранить оригинальный порядок — используйте `[...numbers].sort(...)`.

> [!WARNING]
> **Переполнение при больших n:** при очень большом `n` произведение `n * (n + 1)` может переполнить целочисленный тип. В TypeScript числа 64-битные и safe до `2^53 - 1`, поэтому для ограничений задачи (`n ≤ 10^4`) это не проблема.
